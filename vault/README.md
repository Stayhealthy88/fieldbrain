# FIELDBRAIN 데모 볼트 (가상 훈련 데이터)

> **전량 가상(synthetic)·훈련 시나리오다.** 실제 부대·지명·좌표·교범과 무관하며, 공개·합성 데이터만 포함한다.
> 군사기밀·대외비 정보 없음. 데모/개발 검증 목적 전용.
> **부대명(엑스레이대대)·교범명(경계교범/도하교범(가상))·좌표(TRN 격자)는 실존 대상과 무관한 창작 표식이다. 좌표는 가상 사판 상대값으로 실지형과 무관하다.**

## 시나리오 개요
가상 **엑스레이대대**의 3일간(2026-07-02 ~ 07-04, 예상 D일 07-05) 방어 훈련. 마령고지 일대에 방어진지를 편성하고,
**은하능선 축선의 반복 드론 정찰**과 **청류천/여울목 도하 문제**, 대항군 정찰조 접촉, 보급로 변경에 대응한다. 전 지명·부대·좌표는 창작이다.

## 구조
- `SCHEMA.md` — 볼트 운영 규칙(보고 양식·RBAC·ingest/query/lint 규칙). LLM이 먼저 읽는다.
- `raw/doctrine/` — 가상 예시 교리 3건 (경계작전 / 도하·기상 / 적 공격전술 판단). **실제 교범 아님, 공개출처 일반 원칙.**
- `raw/logs/` — 상황일지 12건 + **대대 방어 작전명령(OPORD, 5개항) 1건** (시간순).
- `raw/comms/` — 수신 보고 원문(SALUTE) 1건.
- `wiki/situation.md` — LLM이 raw로부터 자동 유지보수한 상황판 **예시 출력**(ingest 결과물).

## 데모용 예시 질의
- "은하능선 드론, 교리상 어떻게 판단하고 대응해야 하나?" → 3일 반복 패턴 + 경계교범(가상) §2.3 인용
- "대항군 정찰조 현재 위치는?" → ⚠ 상반 보고 재확인 요구(lint)
- "현재 보급로는?" → 여울목으로 우회 지정됨(최신 작전변경), 백석교 지정은 대체됨. 단 07-04 기상으로 도섭 위험 → 대체수단 검토 중
- "지금 청류천 도섭 가능한가?" → 실측 수심 0.7m + 유속 1.6m/s + 강우 지속 종합, 도섭 재검토 경고
- (소대장 vs 참모 역할) "보급로 변경 상세는?" → RBAC로 접근 차이(작전변경 = clearance 2)

04 CORE IDEA

RAG를 넘어 — 스스로 정리하는 전장 위키

쿼리 시점 검색(RAG)만으로는 흩어진 보고가 지식이 되지 않는다. FIELDBRAIN은 지식을 사전 컴파일하고 지속 유지보수한다 — 보고가 쌓일수록 상황판이 복리로 정확해진다.

RAW불변 기록층 — 기록은 조작되지 않는다

전장 이벤트 로그·수신 보고(SALUTE)·교리를 write-once로 자동 직렬화. LLM은 읽기전용 — 원본 증거가 항상 보존되고, 모든 답은 여기로 소급 가능하다.

WIKI지식층 — LLM이 유지보수하는 상황판

신규 보고 1건 → 상황판(METT-TC)·부대 엔티티·전훈 페이지를 [[크로스링크]]로 갱신. 옵시디언 그래프 뷰가 곧 관계망 시각화 — 지휘소의 세컨드 브레인.

QUERY강제 이중 인용 — 환각의 물리적 차단

모든 답에 [교범 조항 ID] + [보고 타임스탬프] 인용 강제, 근거 없으면 거부. 보고 상충 시 최신 우선(작전변경 > 작전개요), 상충은 명시. RBAC 프리필터로 권한 밖 문서는 검색 자체가 차단.

LINT정보융합 자동화 — 모순은 스스로 찾는다

엔티티 추출(부대·위치·시각)로 보고를 교차검증: 동일 적 부대의 상반된 위치 보고, 낡은 상황판 항목에 ⚠ 재확인 요구. 정보장교의 첩보 교차검증(intel fusion)을 오프라인에서 자동화 — 어떤 선행 프로젝트에도 없다.

INGEST — 상황병

QUERY — 작전참모

근거 조항을 달아 보고한다. 의미 검색(KURE-v1 + LanceDB) + RBAC 프리필터 + 인용 강제·검증(한국어 sLLM · Mi:dm 2.0) — 근거 없으면 답변 거부.

LINT — 정보장교

첩보 교차검증. 엔티티 추출(부대·위치·시각) + 모순·스테일 탐지 → ⚠ 재확인 요구 — 지휘소가 믿어도 되는 지식만 남긴다.

10 DIFFERENTIATORS

비행기모드 라이브 실증 + nettop 외부 연결 0건 + 콜드스타트 리허설. "보장"을 무대에서 증명.

RBAC 통제

검색 전 프리필터 — 권한 밖 문서는 컨텍스트 진입 자체가 불가. 역할 전환 라이브 데모.

저사양 최적화

Q4 + 컨텍스트 캡 + 씬 터미널(오프라인 핫스팟). 1-bit 양자화 로드맵 — 정전 시 배터리 구동까지.

소버린 스택

Mi:dm 2.0(MIT·완전 자체개발) + KURE-v1(고려대·MIT). 국방부 DeepSeek 차단·SKT 국방 MOU와 정합.

그리고 다른 팀에 없는 카드 — "저는 현역 육군 대위입니다. 이 보고 체계로 실제로 근무합니다. CoverMe(오프라인 긴급 대피 경로)에서 이미 골든타임×오프라인 문제를 다뤘습니다."
11 USE CASES · 지휘소의 24시간

C4I 곁의 참모 어시스턴트 — 이렇게 쓴다

ATCIS·B2CS를 대체하지 않는다 — 그 곁에서 기억하고, 근거를 달아 조언하고, 재확인을 요구한다. (아래 시나리오는 전부 합성 데이터)

06:32상황일지 전기 — 훈련·실상황

상황병 일지 입력: "2중대 OP, 미상 소형드론 1대 북동→남서 통과, 고도 약 150m, 채증 실패"

✓ 4초 뒤 질의 가능. 상황판(situation.md) 갱신, [[적-드론]] 엔티티에 3번째 관측 누적 — 일지가 죽은 기록이 아니라 검색 가능한 자산이 된다.

06:40교리 연계 참모조언

지휘관: "드론이 사흘째 같은 시간대에 보인다. 교리상 대응 기준과 권고는?"

"관측 3건 모두 06:30±10분·동일 축선 일지 #41·47·52 — 반복 정찰 징후 시 경계 강화·상급 보고 기준 충족 야교 3-1 §2.3. 조언: 해당 시간대 OP 증강 + 진지 위장 점검."

09:15좌표 × 기상 변수 판단

작전참모: "NK1234 5678 축선으로 우회 기동 — 현재 강우 15mm/h인데 도하 가능한가?"

"해당 좌표는 OO천 하류 [오프라인 지도]. 강우 6시간 지속 시 도섭 기준 수위 초과 예상 도하작전 교범 §4.1. 대안 축선: △△교 경유 +2.4km. ⚠ 기상은 최종 수신 08:00 기준"

21:40상급 보고 초안

통신 일시 복구 — 작전과장: "지난 6시간 상황, 연대 보고용으로 정리해줘."

보고 14건 → 양식에 맞춘 요약 초안 + ⚠ 미해결 재확인 요구 1건 별도 표기, 전 항목 원본 출처 링크 — 참모는 쓰지 않고 검토만 한다. 링크가 살아있는 짧은 순간에 송신까지.

SSD구성 — plug-and-brain

모델(GGUF) + 런타임(포터블 Ollama·단일 실행파일) + 볼트 + 오프라인 지도 = SSD 1개(~30GB). 설치 0 — 지휘소 노트북에 꽂으면 구동.

SSD보안

하드웨어 암호화(지문·키패드) SSD — 볼트가 금고 보관 가능한 물리 통제 자산이 된다. C4I망과 물리 분리, 에어갭 유지.

SSD운용

CP 전환·이동 = SSD만 뽑아간다. 부대 인수인계 = SSD 인계. 파일 기반이라 N개 복제 = 피격 대비 생존성.

12 BUILD PLAN · 지금부터 24H

24시간, 코어 루프에 몰빵

우선순위는 하나 — ingest→query→lint가 심사위원 눈앞에서 도는 것. 부가 기능은 코어가 E2E 2회 완주한 뒤에만.

H0–12 — 코어 루프

  • H0–2 셋업: 모델·임베더 오프라인 캐시 워밍 + 교범 조항 청킹 + SCHEMA.md
  • H2–6 ingest: 볼트 워처(디바운스·삭제 처리) + 증분 색인 + 위키 갱신
  • H6–10 query: 이중 인용 + 검증기 + 거부 게이트 + RBAC 프리필터
  • H10–12 lint: 엔티티 추출(부대·위치·시각) + 모순·스테일 플래그

H12–24 — 데모화

  • H12–15 UX: 단일화면 · 인용 칩 · lint 패널 · 지연시간 표시
  • H15–18 시나리오: 합성 상황일지 12건 + 모순 보고 2건 주입·튜닝
  • H18–20 (코어 E2E 2회 완주 시) 기상·도하 판단 데모 — 교리 기준표 룰 계산
  • H20–22 오프라인 하드닝(CDN 0·nettop) + 콜드스타트 리허설 ×2
  • H22–24 3분 피칭 리허설 + 제출

역할 A — 코어 파이프라인

워처·ingest·query·인용 검증·lint 룰. 리스크: 한국어 BM25(kiwipiepy or 벡터-only 시작).

역할 B — 데모 & UX

단일화면 UI, 인용 칩, lint 플래그 패널, RBAC 토글, 오프라인 하드닝(CDN 0), 데모 스크립트.

역할 C — 코퍼스 & 시나리오

교범 청킹 품질, 합성 상황일지·모순 시나리오, 평가 질의 20개, (여유 시) 기상·좌표 판단 기준표.

13 ROADMAP

노트북 한 대에서 전장 엣지 인프라로

NEXT실단말 포팅 + ATAK 연동 + 저전력

러기드 태블릿·군용 단말 포팅, 1-bit 양자화(정전 시 배터리 구동). 지도 레이어는 ATAK 플러그인/CoT 피드로 — 검증된 생태계에 얹는다.

NEXTLoRa 메시 + 구조화 명령 바이패스

엣지 노드 1대가 분대 단말을 서빙. KR920(LBT) 패킷당 한글 ~66자·전파 2.16초 → 요약 응답 모드 전제. 구조화 입력(@분대 &이동)은 추론 생략 즉시 실행 — 저대역·저전력에 최적. "안 만든 게 아니라, 계산하고 뒤로 뺐다."

NEXT부대별 온보딩
